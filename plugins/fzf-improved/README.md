# fzf-improved

Custom oh-my-zsh plugin built on top of the [oh-my-zsh `fzf` plugin](https://github.com/ohmyzsh/ohmyzsh/tree/master/plugins/fzf).
Configures [fzf](https://github.com/junegunn/fzf) with rich previews, smart directory exclusions,
`fd`-backed search, and in-panel toggle keybindings.

**Load order matters** — `fzf-improved` must come before `fzf` so its environment variables are set
before fzf's shell integration is sourced:

```zsh
# system/bundles.sh
(( $+commands[fzf] )) && plugins+=(fzf-improved fzf)
```

---

## Dependencies

| Tool | Role | Required |
|------|------|:--------:|
| [fzf](https://github.com/junegunn/fzf) | Fuzzy finder engine | ✓ |
| [fd](https://github.com/sharkdp/fd) | Fast file and directory traversal | ✓ |
| [bat](https://github.com/sharkdp/bat) | Syntax-highlighted file previews | optional |
| [lsd](https://github.com/lsd-rs/lsd) | Rich directory tree previews | optional |

`bat` falls back to `cat` and `lsd` falls back to `ls -lh --color=always` when not installed.

## Keyboard shortcuts

### Command line

| Linux | macOS | Action |
|-------|-------|--------|
| `Ctrl+T` | `Ctrl+T` | Fuzzy-find files/dirs and paste selection onto the command line |
| `Ctrl+R` | `Ctrl+R` | Search command history and paste selected entry onto the command line |
| `Alt+C` | `Esc+C` | Fuzzy-find a directory and `cd` into it |

### Inside the fzf panel

| Key | Applies to | Action |
|-----|------------|--------|
| `Enter` | All | Confirm selection |
| `Esc` / `Ctrl+C` | All | Cancel |
| `↑` / `Ctrl+K` / `Ctrl+P` | All | Move cursor up |
| `↓` / `Ctrl+J` / `Ctrl+N` | All | Move cursor down |
| `Tab` | All | Toggle mark (multi-select) |
| `Shift+Tab` | All | Toggle mark and move up |
| `Ctrl+A` | All | Select all visible items |
| `Ctrl+D` | All | Deselect all |
| `Ctrl+/` | All | Toggle preview pane |
| `Ctrl+\` | `Ctrl+T` | Toggle between files and directories |
| `Ctrl+]` | `Ctrl+T`, `Alt+C` / `Esc+C` | Toggle hidden files on/off |
| `Ctrl+G` | `Ctrl+T`, `Alt+C` / `Esc+C` | Toggle gitignored files on/off |
| `Ctrl+E` | `Ctrl+T` | Open selected file(s) in `$EDITOR` (terminal editor) and close fzf |
| `Alt+E` | `Ctrl+T` | Open selected file(s) in `$VISUAL` (GUI editor) and close fzf |
| `Ctrl+Y` | `Ctrl+T`, `Alt+C` / `Esc+C` | Copy absolute path to clipboard and close fzf |

#### Prompt state

The prompt encodes the current mode and active toggles:

| Prompt | Mode | Hidden | Gitignored |
|--------|------|:------:|:----------:|
| `Files> ` | files | off | off |
| `Files +h> ` | files | on | off |
| `Files +g> ` | files | off | on |
| `Files +hg> ` | files | on | on |
| `Dirs> ` | dirs | off | off |

`Ctrl+]` and `Ctrl+G` toggle their flags independently — enabling one does not affect the other.

## Tab completion

Typing `**` before `Tab` triggers fuzzy completion for any command that accepts file or directory
arguments.

| Pattern | Completes |
|---------|-----------|
| `vim **<Tab>` | Files |
| `cd **<Tab>` | Directories |
| `ssh **<Tab>` | Hosts from `~/.ssh/config` and `known_hosts` |
| `kill **<Tab>` | Running processes |
| `export **<Tab>` | Shell variables |
| `unset **<Tab>` | Shell variables |

The candidate list is generated by `_fzf_compgen_path` / `_fzf_compgen_dir` using
`fd --hidden --no-ignore` with `FZF_EXCLUDED_DIRS` applied — dotfiles and gitignored files are
included, but heavy directories (`node_modules`, `.cargo`, `.git`, etc.) are skipped for
performance.

## Configuration

All variables below should be set in `system/environment.local.sh` (loaded before the plugin).

### `FZF_EXCLUDED_DIRS`

Directory names that `fd` skips during all searches (`Ctrl+T`, `Alt+C`/`Esc+C`, `**` completion).
Matching is by name at any depth — `src/node_modules` is excluded the same as a root-level
`node_modules`.

| Category | Directories |
|----------|-------------|
| Language / build / cache | `__pycache__` `.angular` `.cache` `.mypy_cache` `.venv` `.next` `node_modules` `bower_components` `dist` `build` `out` `coverage` `target` `elm-stuff` `.cargo` `.pnp` `.pnp.js` |
| Dev tools & VCS | `.vscode` `.idea` `.git` `.hg` `.svn` `.m2` `.npm` `.nvm` `.sdkman` `terraform.tfstate.d` |
| macOS system | `.DS_Store` `.AppleDouble` `.Spotlight-V100` `.Trash` `.Trashes` `Library` |
| Linux system | `lost+found` `.proc` `.sys` `.dev` `.run` `.var` |

To add exclusions:

```zsh
# system/environment.local.sh
FZF_EXCLUDED_DIRS+=(my-big-cache another-dir)
```

> [!IMPORTANT]
> `FZF_EXCLUDED_DIRS` must be extended before the plugin loads — use an environment file,
> not an aliases or functions file.

### `VISUAL` and `EDITOR`

Two bindings handle editors independently:

| Key | Variable | Intended for |
|-----|----------|-------------|
| `Ctrl+E` | `$EDITOR` (falls back to `vi`) | Terminal editors — opened with TTY redirection |
| `Alt+E` | `$VISUAL` (falls back to `$EDITOR`, then `vi`) | GUI editors — opened without TTY redirection |

```zsh
# system/environment.local.sh
export VISUAL=code   # alt-e
export EDITOR=nvim   # ctrl-e
```

> [!NOTE]
> `Ctrl+Shift+E` is indistinguishable from `Ctrl+E` at the terminal protocol level — the shift
> modifier is lost for control characters. `Alt+E` is used instead.

### Clipboard (`Ctrl+Y`)

The clipboard backend is detected automatically at plugin load time:

| Platform | Tool detected | Fallback |
|----------|--------------|---------|
| macOS | `pbcopy` | — |
| Linux | `xclip -sel clip` | `xsel --clipboard --input` |
| No clipboard tool | `bat` | `cat` |

The absolute path is copied (via `realpath`), with no trailing newline.

### Previews

| Shortcut | Preview |
|----------|---------|
| `Ctrl+T` | Directories: `lsd` tree (depth 2); files: `bat` with syntax highlighting |
| `Alt+C` / `Esc+C` | `lsd` tree (depth 2) |
| `Ctrl+R` | None — uses `--scheme=history` for recency-weighted ranking |
| `**` completion | Same as `Ctrl+T` |

All previews open in a right-side panel at 60% width. `bat` falls back to `cat`; `lsd` falls back
to `ls -lh --color=always`.

### Display (`FZF_DEFAULT_OPTS`)

| Flag | Effect |
|------|--------|
| `--height=60%` | Opens inline at 60% of terminal height |
| `--layout=reverse` | Results at the top, prompt at the bottom |
| `--border` | Draws a border around the widget |

## Internals

### Exclude opts

`_fzf_exclude_opts` is a global array (`typeset -ga`) built at plugin load time by expanding
`FZF_EXCLUDED_DIRS` into `--exclude <dir>` flags. It is used in three forms:

- **String form** (`${_fzf_exclude_opts[*]}`) — embedded directly in `FZF_CTRL_T_COMMAND` and
  `FZF_ALT_C_COMMAND`, which are evaluated later in a subshell where the array is not in scope.
- **Exported string** (`FZF_EXCLUDE_OPTS`) — inherited by `transform:` subshells so they can
  embed it in `reload(...)` actions at runtime.
- **Quoted array form** (`"${_fzf_exclude_opts[@]}"`) — used in `_fzf_compgen_path` and
  `_fzf_compgen_dir`, which run in the current shell and need correct handling of names with
  spaces.

### Toggle bindings

fzf's `transform:` bindings execute in a POSIX `sh -c` subshell — zsh functions are not
available. Toggle logic is exported as a shell snippet in `FZF_IMPROVED_HELPERS` that each
`transform:` binding evals inline before calling `_fzf_toggle`.

All state is encoded in `$FZF_PROMPT` (e.g. `Files +hg> `), which fzf reliably exposes to
`transform:` subshells after every `change-prompt`. Each toggle reads mode and flags from the
prompt, flips one dimension, and emits `reload(...)+change-prompt(...)`. No temp files, no header
parsing.
